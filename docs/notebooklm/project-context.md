# Project Context Bootstrap

Project: DeepValue Lab

Status: NotebookLM bootstrap initialized on 2026-03-15.

This repository is currently an early-stage or placeholder workspace. The NotebookLM project starts with minimal curated context and should be expanded only after real project structure, decisions, and workflows exist.

## What Belongs In NotebookLM

- architecture notes and bounded-context summaries
- validated implementation decisions and tradeoffs
- runbooks, debugging learnings, and recurring failure modes
- external references that the team repeatedly uses
- concise product or workflow context that remains useful across threads

## What Does Not Belong In NotebookLM

- secrets, credentials, API keys, cookies, or tokens
- raw transient logs and uncurated dumps
- anything that conflicts with the current repository state without being clearly marked historical
- speculative notes that were not validated

## Agent Operating Model

1. Check NotebookLM for stable project context.
2. Inspect the repository to confirm the current implementation.
3. Prefer the repository when NotebookLM and code disagree.
4. After validated non-trivial changes, add concise curated updates back into NotebookLM.

## Recommended Next Updates

- add a short project overview once the first real code or product scope exists
- add architecture notes after the initial structure is settled
- add debugging notes after the first real incidents or recurring issues
