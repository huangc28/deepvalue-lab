Implementation rules:
1. Default `mode` to `'legacy'` only when `reports` is `undefined`; otherwise default to `'live'`.
2. Preserve the current interaction rules from Phase 4 and Phase 5:

   - default selection falls back to the first report
   - compare target must differ from the selected base
   - clicking the compare target swaps base and compare
   - clearing or exiting compare does not clear the base selection
3. Support both controlled and uncontrolled usage:
   - if `selectedId` / `compareId` callbacks are provided, use them
   - otherwise keep the current internal `useState` behavior so the component still works before page wiring lands

   - `HistoryLoadingState`
   - `HistoryErrorState`
   - `SelectedRevisionLoadingState`
   - `SelectedRevisionErrorState`
   - `CompareRevisionLoadingState`
   - `CompareRevisionErrorState`
5. Render branches must behave exactly like this:
   - `mode === 'legacy'` renders `LegacyHistory`
   - `mode === 'live'` plus `listState.status === 'loading'` renders `HistoryLoadingState`
   - `mode === 'live'` plus `listState.status === 'error'` renders `HistoryErrorState`
   - `mode === 'live'` plus `reports?.length === 0` renders `EmptyHistoryState`
   - selected-detail loading/error renders inside the right-hand selected snapshot area only
   - compare-detail loading/error renders inside compare mode only and must keep the selected base revision visible
6. Keep the existing fallback-label rule unchanged in both selected and compare cards:
   - `const showFallbackIndicator = locale === 'zh-TW' && report.localeHasFallback`

Do not add fetch calls, React Query hooks, or mock/live source selection logic here. This plan only defines the render contract the page will drive in Plan 02.
