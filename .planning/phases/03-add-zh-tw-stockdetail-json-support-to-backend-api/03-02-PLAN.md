  <action>
Add `be/lib/app/stocks/locale.go` with one small shared helper for this package. The helper should:

- read `r.URL.Query().Get("locale")`,
- return `true` only when the exact value is `zh-TW`,
- return `false` for empty strings or any other locale value.

Use that helper in both handlers.

In `be/lib/app/stocks/detail_handler.go`, implement this exact control flow:

1. Load `row := h.queries.GetPublishedStockDetail(...)` as today.
2. If locale is zh-TW and `row.R2DetailZhTwKey != ""`, download that object and return it.
3. If locale is zh-TW and no zh-TW R2 key exists:
   - if `strings.TrimSpace(row.SummaryJsonZhTw)` is neither `""` nor `"{}"`, return `json.RawMessage(row.SummaryJsonZhTw)`;
   - else continue to EN fallback.
4. EN fallback remains:
   - if `row.R2DetailKey == ""`, return `json.RawMessage(row.SummaryJson)`;
   - otherwise download `row.R2DetailKey` and return it.
5. If EN or zh-TW R2 download fails for the selected branch, log the selected key and return HTTP 500. Do not silently switch to another artifact after a download error.

In `be/lib/app/stocks/list_handler.go`, implement per-row locale selection:

- when locale is zh-TW and `strings.TrimSpace(row.SummaryJsonZhTw)` is neither `""` nor `"{}"`, append `row.SummaryJsonZhTw`
- otherwise append `row.SummaryJson`

Do not change response envelope shape (`{"stocks": [...]}`).
